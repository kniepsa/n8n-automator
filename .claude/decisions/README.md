# Architecture Decision Records

This folder contains Architecture Decision Records (ADRs) for n8n-automator.

## What is an ADR?

An ADR documents a significant architectural decision, including:

- The context and problem
- The decision made
- The consequences (positive and negative)

## ADR Template

Use `000-template.md` as a starting point for new ADRs.

## When to Write an ADR

- Choosing between major technologies
- Architectural patterns that affect the whole system
- Decisions that are hard to reverse
- Trade-offs with significant consequences

## Status Legend

- **Proposed**: Under discussion
- **Accepted**: Implemented
- **Deprecated**: No longer applies
- **Superseded**: Replaced by another ADR
